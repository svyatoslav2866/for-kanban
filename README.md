# for-kanban «Service Exchange — Development Board»


1. Канбан доска в проекте используется для командной разработки, а именно для удобства и наглядного представления всех задач, распределение их между сотрудниками с указанием их роли, приоритета, статуса выполнения и к какой user story относится данная задача, какая фича реализовывается. Вся организационная часть происходит здесь, можно отследить, прошла ли задача проверку и является ли ее статус выполненным, если нет, то необходимо доработать.
2. Backlog - все задачи проекта, To Do — задачи, готовые к работе, In Progress — задачи в работе, Review — проверка / согласование с менеджером проекта и заказчиком,	Done — завершённые задачи.
3. Новая задача создается через нажатие кнопки внизу экрана "Add item".
4. Сначала задача переходит в статус "to do", что означает "гототва к выполнению", дальше ее берет в работу соответствуюее должностное лицо, после выполнения задачи ее статус переходит в "review", где ее согласовывают с заказчиком и отправляют дальше, в противном случае ее отправляют на доработку. В случае, если задача прошла проверку, ее статус становится окончательным "Done" - выполнено.
5. Все задачи распределяюься между ролями в соответствии с должностью и грейдом.
6. Прогресс проекта отслеживается путем демонстрации реализованного функционала заказчику и статусу выполненных задач.
